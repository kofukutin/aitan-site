【null】の意味・使い方・例文 ― voidとの違い【TOEIC英単語】

🔰 nullの発音と意味

[形容詞] 無効の [名詞] ゼロ（可算）

主にITや法律の分野で使われ、値や効力が存在しないことや、無効であることを表します。データベースやプログラミングでは「値が設定されていない」状態を指すことが多いです。

ラテン語の“nullus”（何もない、存在しない）に由来し、接頭辞“null-”が「無」を意味します。英語では16世紀ごろから使われ始め、特にIT分野で広く用いられるようになりました。

✏️ nullの例文

【日常会話】
The result was null.
　結果は無効でした。

This field cannot be null.
　この項目は空欄にできません。

【ビジネス英語】
If the value is null, the system will automatically assign a default setting to the user profile.
　値が空の場合、システムは自動的にユーザープロファイルに初期設定を割り当てます。

🎯 TOEICの出題頻度と傾向

  • 難易度：★★★★☆
  • 出題パート：Part7（文書問題）

ITや契約関連の長文読解問題で、データや効力が「無効」であることを示す語として出題されることが多いです。

❓ クイズに挑戦！

正しい使い方はどっち？

  1. If the input is null, the program will stop running.
  2. If the input is full, the program will stop running.

✅ クイズの答えと解説

  • 正解：1
  • 解説：正解は1です。nullは「値がない」ことを示し、fullは「満杯」を意味するため文脈が異なります。

🤔 void との違いは？

  • null：存在しない、無効である、値が設定されていないという冷静で客観的な印象です。
  • void：voidは「空っぽ」や「無効」という意味で、より法的・公式な場面や抽象的な空間の欠如を強調します。

nullは主にITやデータの分野で「値がない」ことを指し、voidは契約や法律などで「効力がない」「空である」ことを強調します。
